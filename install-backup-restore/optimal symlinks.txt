1) stop les containers

2) casser le symlink actuel sur /DATA+ Créer le nouveau dossier /DATA (rsync est censé le faire mais histoire de.. vu que étape 6 chown autant le faire là)
sudo rm /DATA
sudo mkdir /DATA

3-A) Copier avec rsync de mnt/Storage/DATA vers /DATA mais pas copier mnt/Storage/DATA/Media

OPTION 1:
# Copie TOUT le contenu de /mnt/Storage/DATA/ SAUF le dossier Media 
# vers le nouveau dossier /DATA (sur le SSD)
sudo rsync -avP --exclude 'Media' /mnt/Storage/DATA/ /DATA/
-----
OPTION 2:
#Faire une copie par dossier:
	sudo rsync -avP /mnt/Storage/DATA/AppData /DATA/AppData
	sudo rsync -avP /mnt/Storage/DATA/Documents /DATA/Documents
	etc
-----
	
3-B) Aller voir dans les dossier /DATA que c'est bien comme on veut et pas /DATA/DATA
sinon faut déplacer /DATA/DATA dans /DATA avec la commande mv

4) créer un dossier /DATA/Media (vide, il sera symlink vers le /mnt/Storage/DATA/Media)
sudo mkdir /DATA/Media

5) sudo ln -s /mnt/Storage/DATA/Media /DATA/Media

6) sudo chown -R 1000:1000 /DATA

7) Relancer les containers

8) sudo pihole reloaddns
